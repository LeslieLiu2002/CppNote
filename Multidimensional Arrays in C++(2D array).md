# 内存分配的本质是“指针的指针”
当你写下 `int a2d = new int*[50];` 时，你**并没有分配任何存储整数的内存**。你实际上只分配了能存放 50 个“整数指针”的内存空间（在 32 位系统中仅仅是 200 字节）。为了真正存储数据，你必须再写一个 `for` 循环，为这 50 个指针分别单独分配内层的 `int` 数组。对于三维数组（`int***`），则需要嵌套两层循环来分配。
# 致命的内存泄漏：多维数组的释放规则
- **错误做法**：如果你分配了上述的二维数组，然后直接写 `delete[] a2d;`，**程序不会报错，但会发生严重的内存泄漏**。这行代码只会销毁最外层存储指针的数组，而所有实际存放数据的内层数组依然变成了无主内存。
- **正确做法**：释放内存的顺序必须与分配的顺序完全相反。你必须先写一个 `for` 循环，把内层的 50 个数组逐一释放（`delete[] a2d[i];`），最后再释放外层的指针数组（`delete[] a2d;`）。
# 性能杀手：内存碎片化与缓存未命中 (Cache Miss)
- 由于原生的二维数组内层是由多次不同的 `new` 操作分配出来的，这些内层数组在物理内存中的位置往往是**不连续的（随机分散的）**。这会打破 CPU 缓存（Cache）的连续预读机制，导致缓存未命中（Cache Miss）。
- 建议：尽量**避免使用原生的二维数组**。如果你需要一个 $5 \times 5$ 的二维空间，不要去建 5 个长度为 5 的数组，而是直接建立一个长度为 25 的一维数组：`int* array = new int[5 * 5];`。利用简单的数学公式 **`array[x + y * width]`** 来定位二维坐标即可。
