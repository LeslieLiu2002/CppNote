# 📌 内容总结

## 什么是运算符重载
在 C++ 中，运算符（如 `+`, `-`, `*`, `<<`, `==`）本质上就是函数 。运算符重载允许你为自定义的类或结构体（如自定义的数学向量类 `Vector2`）赋予这些运算符新的具体行为。
> [!tip] 在C++中，对于双目运算符（如`A+B`或者`A<<B`），如果把它重载为类的成员函数，编译器会默认**左边的操作数（`A`）是调用该函数的对象**。
  
**语法**：定义形式类似于普通的函数，只需要将函数名替换为 `operator` 关键字加上对应的运算符符号（如 `operator+`）。
## 重载算术运算符
```c++
struct Vector2
{
	float x,y;
	vector2(float x,float y):x(x),y(y){}
	
	Vector2 operator+(const Vector2& other) const
	{
		return Vector2(x+other.x,y+other.y);
	}
};
```
我们写`vec1 + vec2`，底层实际上是`vec1.operator+(vec2)`。
## 重载输出流运算符 (`<<`)
编译器默认左边的操作数是调用该函数的对象。而`std::cout`是`std::ostream`类的实例化对象，但我们无法修改C++官方标准库的源码，因此解决方案：写出非成员（全局）函数
```c++
//在类外
std::ostream& operator<<(std::ostream& stream, const Vector2& other)
{
	stream<<other.x<<","<<other.y;
}
```
> [!info] 为什么可行?（调用`std::cout<<myVector;`）
> 编译器首先会尝试把它当作成员函数调用，翻译为`A.operator<<(B)`，但发`ostream`库中没有这样的定义，于是尝试把它当作全局函数调用，翻译成`operator<<(A,B)`，成功找到定义。（个人理解：因为类其实就是作用域/命名空间，而全局也是一个作用域/命名空间，所以全局也是一个类，那这里其实可以理解成`全局类.operator<<(A,B)`）
## 重载下标运算符
```c++
class String
{
private:
	char* m_Buffer;
...
public:
	char& operator[](unsigned int index)
	{
		return m_Buffer[index];
	}
};

String s="Leslie";
s[2]='a';
```
因为下标运算符被定义为了一个特殊的“环绕式”运算符，`()`也是，它们被设定为去括号中寻找参数。

# ⚠️ 需要注意的重难点（重点与易错点）

1. **代码可读性与滥用问题 (非常重要)**
    - 重载运算符的初衷是**简化代码并增强直观性**。只有当使用运算符极其符合直觉（例如数学向量的加减）时，才应该重载它。千万不要强行为了“炫技”去滥用运算符重载（例如用 `&` 去给对象赋值），否则会导致代码的意图非常晦涩，极难阅读和维护。
2. **运算符优先级 (Operator Precedence)**
    - 重载后的运算符仍然遵循 C++ 内置的运算符优先级规则。例如，在一个包含重载后加法和乘法的表达式中，乘法依然会先于加法被计算。
3. **提供函数替代方案（API 设计建议）**
    - 建议，在编写库或者 API 时，除了提供重载后的运算符外，最好同时也保留带有明确命名的普通函数版本（如既有 `operator+`，又有 `Add()`），从而将选择权交给 API 的使用者。
