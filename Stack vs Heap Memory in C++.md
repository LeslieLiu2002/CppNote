> [!info] 两者存储的物理位置相同
 栈和堆虽然名字和机制不同，但它们在物理层面上都位于计算机的随机存取存储器（RAM）中。

- **容量限制**：栈通常具有预设的固定大小（通常约为2MB），而堆的大小虽然也有默认值，但可以随着应用程序的运行动态增长 。
- **代码层面的分配与释放**：
    - **栈**：通过直接声明变量分配（如 `int value = 5;`）。当变量离开其所在的作用域（Scope）时，内存会被自动释放。
    - **堆**：需要使用 `new` 关键字（或智能指针）来进行分配。在堆上分配的内存必须由程序员使用 `delete` 关键字手动释放，否则会导致内存泄漏。
- **分配性能对比**：
    - 栈分配极快，本质上只需要移动一次栈指针（大约只需一条CPU指令）即可完成内存分配。
    - 堆分配较慢，它通常会调用 `malloc` 等底层函数，并在“空闲列表”（Free List）中寻找合适的内存块，需要进行大量的记录和管理（Bookkeeping）。如果内存不足，还需要向操作系统请求更多内存，这一过程非常消耗性能。
    - 两者最核心的性能差异是在于**内存分配**的阶段。栈分配几乎没有开销（仅需移动指针），而堆分配涉及复杂的寻址和底层系统调用。
