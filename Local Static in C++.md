> [!info] 和类中的static类似，相当于函数中的共有量
# 基本概念：
- **生命周期（Lifetime）**：局部静态变量的生命周期与整个程序的运行周期相同。        
- **作用域（Scope）**：其作用域依然被限制在声明它的局部范围内（例如函数内或 `if` 语句内），在范围之外无法被直接访问。     
# 基本行为测试：
```c++
void Function()
{
	static int i = 0;
	i++;
}
Function();//i=1
Function();//i=2，i没有被重新初始化为0
Function();//i=3，i没有因为超出作用域而被销毁
```
- 在函数内部声明一个 `static int i = 0;`。当该函数==**第一次**被调用时，变量 `i` 会被初始化==。        
- 随后再次调用该函数时，==**不会**重新创建或初始化该变量==，而是直接引用之前已经存在的那个变量。        
- 局部静态变量的**生命周期是全局的**，但**可见性（作用域）是局部的**。不能因为它的生命周期长，就尝试在函数外部直接通过变量名去访问它。如果确实需要外部访问，只能通过函数返回其引用或指针。
# 实际应用案例：单例模式（Singleton）：
- 传统的单例实现通常需要在类内声明静态指针/实例，并在类外进行定义和初始化，代码较为冗长。        
- 利用局部静态变量，可以直接在 `Get()` 方法中声明一个 `static Singleton instance;` 并返回它的引用。这样不仅代码大大简化，而且能够确保实例在第一次调用 `Get()` 时才被正确构造，且整个程序生命周期内仅此一个。
```c++
class Singleton
{
public:
	static Singleton& Get()
	{
		static Singleton instance;
		return instance;
	}
};
```
