# 修补Const修饰类方法
出于某些原因，比方说debug，我们需要对不是真正意义上属于类的变量进行修改，比如说被debug了多少次，此时将这个类成员修饰为`mutable`就可以在`const`修饰的类方法里修改了
# Lambda
==这种情况在实践里基本不会发生，只存在理论==
看下面的例子：
```c++
int x = 8;
auto f = [=]() mutable
{
	x++;
}
f();// x= 8
```
- `[=]`表明该函数是拷贝传值，所以调用完`f()`后`x`值没变
- 在函数名后加上`mutable`关键字表示能修改传入的变量。如果不加，那么`x++`会报错
