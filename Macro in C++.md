# 宏的本质与预处理阶段
C++ 中的宏是通过预处理器（Preprocessor）来实现的。在编译器进行实际编译之前，预处理器会先进行一次处理。宏本质上就是一个“**纯文本的查找与替换**”过程 。
# 带参数的宏
宏不仅能进行简单的静态替换，还可以接收参数（例如 `#define LOG(x) std::cout << x << std::endl`），从而动态地替换传入的内容。
# 实际应用-日志系统与配置控制
宏在实际开发中最经典的用法是根据构建配置（Debug 还是 Release）来控制代码的编译。通过 `#ifdef` 或 `#if`(使用`#if`时需要给宏赋值，然后在`#if`后用`==`判断) 等预处理指令，你可以在 Debug 模式下保留所有日志输出，而在 Release 模式下直接将其替换为空，从而彻底移除这些日志代码，实现性能优化和信息保密。
# 多行宏定义
如果宏的内容非常长，需要换行，可以在每行末尾使用反斜杠 `\` 来转义换行符，从而将多行代码整合成一个宏。**反斜杠的后面绝对不能有空格**，不然转义的不是换行符而是空格。
> [!tip] 通常不在宏定义的末尾加上分号。
