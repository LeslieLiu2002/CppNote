# 创建静态库
> [!info] 以“Game”为主程序，以“Engine”为静态库

1. 正确设置项目配置类型：
	- 在一个解决方案（.sln）主应用程序项目（Game）的配置类型必须是应用程序（.exe）。
	- 库项目（Engine）的配置类型必须是静态库（.lib）。记得将它的设置应用于“所以配置”(All Configurations)和“所有平台”(All Platforms)
2. 头文件包含路径(Include Paths)的设置:
	- **相对路径的劣势：**当然可以直接用`#include"相对路径"`但一旦移动了文件夹或者重命名项目就会报错
	- **使用宏配置绝对逻辑（推荐）：**更好的做法是进入主项目的属性页（C/C++ -> 常规 -> 附加包含目录），利用 Visual Studio 提供的宏来设置绝对路径。例如使用 `$(SolutionDir)Engine\src`。这样主项目代码中就可以直接使用 `#include "engine.h"`，使代码更清晰且易于维护。
3. 自动化链接静态库：
	- 不建议先build库项目然后手动去链接器里添加生成的`.lib`文件路径[[Using Libraries in C++ (Static Linking)]]
	- **添加引用（Add Reference）**：最简便的方法是在主项目（Game）上右键选择“添加” -> “引用”，然后直接勾选同一个解决方案下的库项目（Engine）。让Visual Studio自动管理链接
4. 理解静态链接的最终产物：由于使用的是静态链接，库项目（Engine）编译出的二进制代码在链接阶段会被完全“打包”并嵌入到最终的应用程序中。这意味着最终部署时，你只需要那一个单独的 `.exe` 文件即可运行，不需要附带任何其他外部文件（没有 `.dll` 依赖）。
