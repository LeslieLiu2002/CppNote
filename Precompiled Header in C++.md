> [!tips] 一些注意点
> 1. 在预编译头文件中放入几乎不会修改的外部库，比如 C++ 标准库 (`<vector>`, `<map>`, `<algorithm>`)、操作系统的 API (`<windows.h>`)、或是庞大且稳定的第三方框架。绝对不能放你自己正在开发、经常修改的头文件。因为预编译头文件稍有变化就要重新编译。
> 2. 在使用 PCH 时，你必须在你所有的 `.cpp` 源文件的**第一行**写入 `#include "pch.h"`。如果你把它放在别的 include 后面，或者写了些代码之后再 include，很多编译器（尤其是 MSVC）会直接报错并忽略它。
# 为什么需要预编译头文件
在 C++ 中，编译时间常常是开发者的一大痛点。当你包含标准库（如 `<vector>`, `<string>`, `<iostream>`）或大型第三方库（如 Windows API）时，编译器会在每个包含这些库的源文件（.cpp）中，将动辄几十万行的头文件代码全部复制进来并从头解析、编译一次。如果你的项目里有 10 个源文件都 include 了 `<vector>`，那么 `<vector>` 就被毫无意义地编译了 10 次。
# 预编译头文件的作用
- 预编译头文件（==通常命名为 `pch.h` 或 `stdafx.h`==）允许你将那些**庞大且不经常变动**的头文件集中放进去。
- 流程：编译器会把这个 `pch.h` 一次性编译成一种二进制格式，这些数据与**源代码文本**等价，但已经是**半编译状态**——经过了词法、语法、语义分析，但**尚未生成中间代码**[[How the C++ Compiler Works#编译]]（注意不是直接生成了机器码）。之后，你的项目在编译其他源文件时，只要直接读取这个二进制文件就可以了。它们和被分析完后的源文件一起被编译为中间代码，然后中间代码被编译为机器码。这样省去了重复的文本解析。
# Visual Studio 下的配置
在 Visual Studio 里手动配置 PCH 的步骤如下：
- 你需要在**项目**(Project)属性（不是解决方案(Solution)）中找到`C/C++->Precompiled Headers->Precompiled Header`然后开启 `Use`（使用预编译头），并将`C/C++->Precompiled Headers->Precompiled Header File`指定头文件的名字（比如 `pch.h`）。
- **关键一步**：你必须为对应的 `pch.cpp` 文件（里面只有一行 `#include "pch.h"`）**单独**右键进入属性，然后选择`C/C++->Preprocessor`，接着将`Precompiled Header`设置改为 `Create`（创建预编译头）。
- 如果你不单独设置 Create，编译器会一直提示找不到预编译的二进制文件。
___
用`GCC / Clang`预编译头文件指令`g++ -std=c++11 pch.h`执行了该指令后会生成一个名为`pch.h.gch`的文件。之后正常编译其他源文件就行了
