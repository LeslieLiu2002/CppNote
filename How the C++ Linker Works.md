链接到主要重点是找到每个字符和函数的位置并且将它们连接在一起，具体表现为找到==所有==符号（函数名、变量名）——不管它们在哪个对象文件里，并将这些独立的对象文件及需要的外部库链接在一起
>[!tip]- compile只编译不链接，build既编译又链接
>- 编译会检查语法错误，检查有没有声明函数(但不管函数定义)
>- 链接会检查是否有entry point（例如main()函数），查找函数定义

# 常见链接错误
>[!tip] 两个函数完全相同或者声明和定义匹配时要满足函数名、参数列表已经返回类型都相同
### 未解析的外部符号(Unresolved External Symbol)
**出现情况：** 任意代码中声明并调用函数，但链接器在==所有==对象文件里都找不到具体定义时
**特殊情况：** 源文件中只声明函数但不调用函数就不会触发链接器的寻找定义动作
## 符号重复定义(Multiply Defined / Duplicate Symbols)
**出现情况：** 当链接器找到两个及以上的完全相同的函数时会产生歧义
**常见陷阱：** 头文件同时声明和定义函数(只有定义时会默认同时也有声明)并且被多个源文件包含
**解决方法：** 
- 头文件只声明，定义放在单独源文件里
- 在头文件里的函数(声明和定义都要)前加上static进行内部链接([[Static in C++#类外Static]])，意味着它只作用于当前文件
- 使用inline标记为内联函数(声明和定义都要)
# 链接外部库(Library)
- **静态链接**：将库的代码直接编译进最终的 `.exe` 可执行文件中。优点是运行速度通常更快（编译器可以做链接时优化），且运行时不需要额外的 `.dll` 文件。在**编译时**库代码会直接打包进最终的可执行文件。[[Using Libraries in C++ (Static Linking)]]
- **动态链接**：在程序运行时才加载库（`.dll` 文件），需要确保运行时环境中存在对应的动态链接库。在**运行时**才会把库代码加载到内存中。[[Using Dynamic Libraries in C++]]
