在类或者结构体外的static：链接性为内部链接，只有定义它的翻译单元可见
在类或者结构体内的static：该变量或方法与所以实例共享内存
# 类外Static
静态变量或函数意味着当真正需要将它们链接起来以定义符号时，链接器不会在这个翻译单元的作用域之外查找该符号的定义。
> [!tip] 也可以用`extern`关键词来修饰符号去外部寻找定义
> 但也是从内开始向外找,并且最好不要初始化

当static出现在局部作用域{}中，例如函数体中[[Local Static in C++]]
# 类内Static
## 静态变量
- 全局唯一性：无论创建了多少个实例，静态变量在内存只有一份
- 跨实例共享：修改静态变量，其他实例看到的这个变量的值也会改变
- 如何访问：类名+作用域解析运算符(`::`)或者实例(`e.X`)，但最好不要用实例的方式
- 必须在外部定义：如果不初始化静态变量，就必须在类外和函数外定义它，以便链接器可以找到它。
```c++
struct Entity
{
	static int x,y;
}

int Entity::x; // 必须要在类外定义
int Entity::y; // 必须在函数外，main函数内也不行

int main()
{
	entity e;

    entity::x=3;
    entity::y=4; // 推荐使用该访问方法

    e.x=5;
    e.y=6; // 不推荐这样访问
	return 0;
}
```
## 静态方法
- 无需实例即可调用：静态方法不依赖于类的任何实例。`Entity::Print()`
- **无法访问非静态成员**：静态方法内部不能访问类中的非静态变量（实例变量），也不能调用非静态方法。
	为什么：普通的非静态类方法在底层编译后，实际上会隐式接收一个指向当前实例的指针(`this`)，而==静态方法没有这个隐藏的实例指针==。
- 可以访问静态成员：它们都在同一个命名空间并且全局唯一
