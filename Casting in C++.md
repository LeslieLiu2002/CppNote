# C风格转换
语法极其简单，比如 `(int)value`。它非常直接，但本质上是“暴力强转”，编译器会尽最大努力按你的意思转，哪怕逻辑上是错的。
# C++风格转换
**基本语法:** `A a = XX_cast<B>(b);`(`A`和`B`是两个不同的类型，`a`和`b`是这两个类型的变量)
C++ 引入了 4 个专门的转换操作符，它们不仅在语法上更长，而且各自有非常明确的职能：
- **`static_cast`**: 最常用的基础转换（如 `float` 转 `int`）。它会在**编译时**进行基本的类型检查。如果两种类型牛头不对马嘴（没有任何隐式转换规则），编译器会直接报错，比 C 风格安全。
- **`reinterpret_cast`**: 底层转换，用于 **类型双关 (Type Punning)**[[Type Punning in C++]]。它直接把这块内存按照新的类型重新解释，不做任何数据转换。极度危险，但非常灵活。
- **`dynamic_cast`**: 动态转换，专门用于**继承体系中的向下转型 (Downcasting)**[[Inheritance in C++#继承中的转型]]。它会在**运行时**检查真正的对象类型（基于 RTTI），要求被转换的类型的类层次结构中**至少有一个虚函数**，如果类型不匹配（比如你试图把一个A类型的指针转换成一个B类型的指针，而类B中有类A中没有的成员，且该指针指向的A类型的对象），它会返回 `nullptr`（空指针），从而防止程序崩溃。如果类型匹配（比如该指针指向B类型的对象），就会得到转换后的指针
- **`const_cast`**: 专门用来**移除（或添加）对象的 `const`（常量）属性**。
