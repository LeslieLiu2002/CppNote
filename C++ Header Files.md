# 为什么需要头文件
因为==编译机制==规定在编译时调用一个函数时需要检查它的声明是否存在，所以一个函数或者变量可以被“声明”多次，但只能“定义”一次。头文件就是一个专门存放声明（函数声明、类型定义如结构图）的通用场所。`#include`将这些声明复制粘贴到需要的源文件中,由预处理器完成
___
# 头文件守卫(Header Guards)
如果在一个复杂项目中，文件互相包含，很容易导致一个头文件被多次包含进同一个源文件中。这会导致“重复定义错误”。
**解决方案**
1. `#pragma one`这个文件在每一个 translation unit 中只允许被包含一次
2. `#ifndef`具体写法如下：
```c++
#ifndef LOG_H
#define LOG_H
// content
#endif
```

# 尖括号`< >`和双引号`" "`的区别
尖括号`< >`：主要用于包含编译器设置的系统包含目录中的文件
双引号`" "`：主要用于包含相对路径的文件。编译器会优先在当前文件所在同级目录进行查找
