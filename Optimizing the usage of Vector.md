# 为什么要优化
- 当容量不足时，`vector`会在内存中开辟一块更大空间，然后把就元素全部拷贝过去在删除旧内存
- 使用`push_back()`时，实际上是现在栈上创建了对象，然后把这个对象拷贝到`vector`的堆内存中
- 拷贝的过程是极其消耗性能的
# 策略
- 使用`.reverse()`预分配内存：可以提前设定好`vector`的容量
- 使用`.emplace_back()`替代`.push_back()`：`emplace_back()`允许我们直接传入对象的构造参数，然后在分配好的空内存上构造这个对象，省去从外部栈拷贝到内部堆的过程
# 易混点
- `std::vector<T> v(3)`与`v.reserve(3)`：前者是在定义的时候会分配3个元素的内存然后调用默认构造函数创建3个对象在这写内存上；后者只会分配能容纳3个元素的内存但不创建对象
