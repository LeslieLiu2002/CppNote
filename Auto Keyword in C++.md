# 基本概念
在C++中，`auto` 关键字允许编译器在编译时根据变量的初始化表达式自动推导出它的数据类型。例如，如果你写 `auto a = 5;`，编译器会推导为 `int`；写 `auto a = 5.5f;`，则推导为 `float`。
> [!tip] 编码规范建议
> 对于简单的基础数据类型（如 `int`、`float`、`std::string` 等），直接写明具体类型更好，因为这能让看代码的人一目了然。建议在应对**非常冗长或复杂的类型**时才会使用 `auto`，或者使用 `using` 或 `typedef` 来取别名。
# 重难点
## 引用与拷贝
- 当一个函数返回一个引用类型（如返回一个大型容器的成员）时，如果你仅用 `auto` 来接收，**编译器默认会进行拷贝，而不是保留引用**
- 正确做法是写成`auto&`，如果只读引用，则应该写成 `const auto&`
## 隐式的API变动风险
- 如果在复杂的项目中使用 `auto` 接收函数返回值，一旦底层API的返回类型被修改（例如从 `std::string` 变成了 `char*`），使用 `auto` 的那行代码本身不会报错。
- 然而，如果你在后续代码中使用了该类型的特有方法（比如调用了原本 `std::string` 才有的 `.size()`），此时就会产生难以排查的编译甚至运行错误。显式声明类型可以帮助你在第一时间把类型不匹配的错误暴露出来。
