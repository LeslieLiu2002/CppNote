# 什么是Type Punning
**Type Punning（类型双关）** 是一个术语，指的是在 C++ 中，**将同一块内存数据按照不同的类型去解释和读取**。（可以告诉编译器：“我知道这块内存原来存的是 `int`，但现在请把它当作 `double` 或者 `数组` 来处理。”这时，编译器从按照`int`指针读四个连续字节变成按照`double`指针读8个连续的字节）
# 语法实现（C风格）
实现 Type Punning 的标准动作（C 风格转换）： 
`获取地址 (&) -> 转换为新类型指针 ((Type*)) -> 解引用取值 (*)` 
例如：`*(int*)((char*)&entity + 4)` （结构体`entity`中有两个`int`变量`x`和`y`，该例子把地址转为 char* 以便按单字节偏移，往后移动 4 字节，再转回 int* 解引用，从而硬核获取第二个变量 `y`）。实际现代 C++ 规范中更推荐使用 `reinterpret_cast`
# 结构体内存对齐
- `Entity` 结构体转 `int[2]` 能成功，是因为结构体`entity`中的`int x; int y;` 在内存中是完全连续的 8 字节。
- **注意**：在实际开发中，如果结构体内部有不同大小的类型（如 `char` 和 `int` 混用），编译器会自动插入**内存对齐的填充字节（Padding）**。如果在有 Padding 的结构体上强行做数组的双关转换，读取的数据位将会错乱。
