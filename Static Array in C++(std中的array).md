# 基本语法
- 声明 `std::array` 时，必须通过模板参数同时指定**数据类型**和**数组大小**，例如 `std::array<int, 5> data;`
- **注意**：这个大小（如这里的 5）必须是一个**常量表达式**（数组大小必须在编译时确定）。如果你需要动态调整大小，必须使用 `std::vector`。
> [!tip] 由于 `std::array` 是一个完整的对象类，传递给函数时，务必按引用传递
> -  _错误示例_：`void printArray(std::array<int, 5> arr)`
> - _正确示例_：`void printArray(const std::array<int, 5>& arr)`
# vs传统C风格数组
- `std::array` 的大小是作为模板参数存在的，它在内存布局上和 C 数组一模一样，**没有任何额外的内存或性能开销**。
- 使用传统的 `data[i]` 访问元素时，性能等同于传统数组。但如果使用 `data.at(i)` 方法，`std::array` 会执行**边界检查**，如果索引越界会抛出异常。
- 传统C风格数组不知道自己的大小，而且在传递给函数时会退化为指针（Pointer Decay）。`std::array`提供了现代 C++ 容器的标准接口（例如内置的 `.size()` 方法可以随时获取数组大小，且支持迭代器）。
# vs`std::vector`
- `std::vector` 是动态数组，会在堆（Heap）上动态分配内存；而 `std::array` 的内存是直接分配在栈（Stack）上的。
- **注意**：操作系统的栈空间通常是有限且较小的（一般为几 MB）。如果你尝试声明一个尺寸极其庞大的 `std::array`（例如包含几百万个元素的数组），程序大概率会触发栈溢出（Stack Overflow）导致崩溃。对于超大数据集，依然需要使用 `std::vector` 将数据分配在堆内存中。
